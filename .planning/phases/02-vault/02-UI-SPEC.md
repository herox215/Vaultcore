# Phase 2 — UI Design Contract

> Visual and interaction contract for the Vault phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS v4 + CSS variables) |
| Preset | not applicable — Svelte 5 project; shadcn/ui is React-only |
| Component library | none — bespoke Svelte components |
| Icon library | lucide-svelte (add in Phase 2 — Phase 1 UI-SPEC §Design System noted "add lucide-svelte in Phase 2 when sidebar tree lands") |
| Font | system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif (OS native stack — no web font fetch, preserves SEC-01 zero-network constraint) |

**Icon usage:** lucide-svelte icons are used exclusively in the sidebar tree (file/folder/chevron) and tab bar (close button X). No icons outside those two surfaces in Phase 2. Size: 16px (`size={16}`) for tree icons, `stroke-width={1.5}` for visual consistency with Phase 1's flat style.

**Source:** Phase 1 UI-SPEC §Design System. CLAUDE.md SEC-01 (zero network calls).

## Spacing Scale

Inherited from Phase 1 — no changes. All tokens carry forward unchanged.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, tight inline spacing |
| sm | 8px | Button inner padding (vertical), list item gap, sidebar row vertical padding |
| md | 16px | Default element spacing, card padding, sidebar horizontal padding |
| lg | 24px | Section padding, dialog padding |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level centering |

**Phase 2 exceptions:**

- Sidebar tree rows: 28px min-height (6px top + 6px bottom padding on 16px body text) — denser than Phase 1's file rows to accommodate deep folder nesting without excessive vertical scroll
- Tab bar height: 36px (8px top + 8px bottom padding on 14px tab label + 2px active-tab indicator bottom border)
- Tree indentation: 16px per nesting level (folder icon column + left padding)
- Split divider: 4px wide drag handle (not a spacing token — it is a component boundary)
- Inline rename input: 24px min-height to match row height without layout shift

**Source:** 8-point grid standard. Phase 1 UI-SPEC §Spacing Scale (inherited). Phase 2 exceptions derived from tree/tab density requirements.

## Typography

Inherited from Phase 1 — no new sizes or weights added. All Phase 2 components use the existing 4-size, 2-weight system.

**App-chrome typography:**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Sidebar tree filenames, tab labels, dialog body text, toast messages |
| Label | 12px | 400 | 1.4 | Sidebar section headers, tree path tooltips, tab dirty-dot aria label |
| Heading | 20px | 700 | 1.2 | Dialog headings (rename confirmation, delete confirmation) |
| Display | 28px | 700 | 1.1 | Not used in Phase 2 (reserved Phase 5) |

**Weights:** 2 weights — 400 (all body/label text) and 700 (dialog headings only).

**Sidebar tree filename truncation:** Single-line, `text-overflow: ellipsis`, `overflow: hidden`, `white-space: nowrap`. Truncates at the end (preserves the start of the filename). Full path shown in tooltip on hover (`title` attribute).

**Tab label truncation:** `max-width: 160px`, single-line, ellipsis at end. Tooltip shows full filename on hover.

**Editor typography:** Unchanged from Phase 1 — CM6 heading sizes (H1 26px, H2 22px, H3 18px), editor body 15px, inline code 13px monospace. Phase 2 adds `readonly` prop to CMEditor but does not change typography.

**Source:** Phase 1 UI-SPEC §Typography (fully inherited). No upstream artifact introduces new type roles for Phase 2.

## Color

All color tokens are already declared in `src/styles/tailwind.css` and carry forward unchanged. Phase 2 introduces no new CSS variables.

```css
/* src/styles/tailwind.css — existing tokens, unchanged */
:root {
  --color-bg:        #F5F5F4;  /* dominant surface */
  --color-surface:   #FFFFFF;  /* cards, editor pane, panels */
  --color-border:    #E5E5E4;  /* dividers, input borders */
  --color-text:      #1C1C1A;  /* primary text */
  --color-text-muted:#6B7280;  /* secondary text, sidebar labels */
  --color-accent:    #6D28D9;  /* purple — reserved elements only */
  --color-accent-bg: #EDE9FE;  /* accent surface (hover states) */
  --color-error:     #DC2626;  /* error toast */
  --color-warning:   #D97706;  /* conflict toast */
  --color-success:   #059669;  /* clean-merge toast */
}
```

| Role | CSS Variable | Hex | Usage |
|------|-------------|-----|-------|
| Dominant (60%) | `--color-bg` | `#F5F5F4` | App window background, sidebar background |
| Secondary (30%) | `--color-surface` | `#FFFFFF` | Editor pane background, tab bar background, dialog surface, tooltip background |
| Accent (10%) | `--color-accent` | `#6D28D9` | See reserved list below |
| Destructive | `--color-error` | `#DC2626` | Delete confirmation button, error toast left-border and icon |
| Semantic: warning | `--color-warning` | `#D97706` | Conflict toast left-border and icon only |
| Semantic: success | `--color-success` | `#059669` | Clean-merge toast left-border and icon only |

**Accent (`--color-accent`) reserved for:**
1. Active/selected tree row in the sidebar (left border 2px `--color-accent`, background `--color-accent-bg`)
2. Active tab in the tab bar (bottom border 2px `--color-accent`, text `--color-text` at weight 700)
3. Inline rename input focus ring (2px outline `--color-accent`, 2px offset)
4. CM6 editor cursor and selection highlight (inherited from Phase 1)
5. Nothing else in Phase 2

**Sidebar background:** Uses `--color-bg` (`#F5F5F4`) — one shade darker than the editor pane (`--color-surface` `#FFFFFF`). This creates the Obsidian-style visual separation between navigation and content without introducing a new color token.

**Split divider:** `--color-border` (`#E5E5E4`) at 1px, hover state widens visual hit area with `--color-accent-bg` background fill at 4px.

**Readonly mode (ERR-03 vault unmount):** All open editors gain a `pointer-events: none` overlay with `background: rgba(255,255,255,0.6)` over the CM6 surface. No new color variable — composed from existing `--color-surface` with opacity.

**Source:** Phase 1 UI-SPEC §Color (fully inherited). CONTEXT.md D-02, D-14.

## Component Inventory

### Layout: VaultLayout (`src/components/Layout/VaultLayout.svelte`)

**Replaces:** `VaultView.svelte` from Phase 1.

**CSS Grid layout:**
```
[sidebar] [divider] [editor-area]
```
- Sidebar: `width: var(--sidebar-width, 240px)` — CSS variable driven by drag handle. Persisted to app settings. Range: 160px (min) to 480px (max).
- Divider: 4px wide, `background: var(--color-border)`, cursor `col-resize` on hover with `--color-accent-bg` fill.
- Editor area: `flex: 1 1 0` — fills remaining space. When split active, uses nested `display: flex; flex-direction: row` with two `EditorPane` instances separated by another 4px divider.

**Split pane sizing:** Draggable split divider, not fixed 50/50. Initial split is 50/50 on first drag. Drag range: each pane minimum 240px. CSS variable `--split-ratio` (default `0.5`) drives `flex-grow` allocation.

**Sidebar collapsed state:** `width: 0`, `overflow: hidden`. Toggle button persists as a 32px × 32px icon button at the left edge of the editor area. Collapses/expands via CSS transition `width 200ms ease`.

### Sidebar (`src/components/Sidebar/Sidebar.svelte`)

**Outer container:** Full height of VaultLayout sidebar column. `display: flex; flex-direction: column`. Background `--color-bg`.

**Header strip (top):**
- Height: 40px
- Vault name: Body (14px/400), `--color-text`, truncated with ellipsis
- Action buttons: "New file" icon button (lucide `FilePlus`, 16px) + "New folder" icon button (lucide `FolderPlus`, 16px). Right-aligned. 32px × 32px touch target. Hover: `--color-accent-bg` background, `--color-accent` icon stroke. No text label in Phase 2 — icon only with `aria-label`.

**Tree area:** Scrollable `overflow-y: auto`, `flex: 1 1 0`. Scroll bar uses OS default — no custom scrollbar styling in Phase 2.

**Sidebar section header (none in Phase 2):** The tree renders directly without a "FILES" label in the sidebar. Section headers are a Phase 5 enhancement.

### Sidebar Tree Node (`src/components/Sidebar/TreeNode.svelte`)

**Row anatomy (left to right):**
1. Indentation: `padding-left: calc({depth} * 16px + 8px)` — 16px per nesting level, 8px base
2. Chevron icon: lucide `ChevronRight` (16px) rotated 90° when expanded, hidden for files (replaced by 16px spacer). Transition: `transform 150ms ease`.
3. File/folder icon: lucide `Folder` / `FolderOpen` (16px, fill for open) for directories. lucide `FileText` (16px) for `.md` files. lucide `File` (16px) for all other file types. `stroke-width={1.5}`. Color: `--color-text-muted` by default, `--color-accent` when row is active.
4. Filename: Body (14px/400), `--color-text`. Truncated with ellipsis. Full path in `title` tooltip.
5. Right-side actions (visible on row hover only): lucide `MoreHorizontal` (16px) button — triggers context menu with: "Rename", "Delete", "New file here", "New folder here". Hover target: 24px × 24px. `--color-text-muted`, hover `--color-text`.

**Row height:** 28px min-height. `display: flex; align-items: center; gap: 4px`. 6px top + bottom padding.

**States:**
- Default: background transparent
- Hover: background `--color-accent-bg`
- Active (selected): left border 2px `--color-accent`, background `--color-accent-bg`, filename color `--color-text` at weight 700
- Disabled/symlink: same as default with `opacity: 0.7` on icon; no pointer cursor on expand
- Drag-over (drop target): dashed 1px `--color-accent` border, background `--color-accent-bg`
- Dragging (source row): `opacity: 0.5`

**Expand/collapse:** Clicking chevron or folder row (not filename) toggles. Keyboard: Space/Enter on focused row.

### Inline Rename (`src/components/Sidebar/InlineRename.svelte`)

**Replaces filename text with an `<input type="text">` in place.**

**Input styling:**
- Same width as the row's filename area
- 14px/400, `--color-text`
- Background `--color-surface`, border 1px `--color-border`
- Focus ring: 2px outline `--color-accent`, 2px offset
- Height: 24px (matches row text height, no layout shift)
- Border-radius: 3px

**Behavior:**
- On mount: pre-fill with current filename, select-all
- Enter: confirm rename, dismiss input
- Escape: cancel, restore original name, dismiss input
- Blur (click outside): confirm rename (same as Enter)
- Validation: if filename contains `/` or `\`: show inline error text "Filename cannot contain / or \\" in `--color-error` at 12px below the input. Do not dismiss. Extension: `.md` is appended automatically if the user omits it (only for `.md` files). Existing non-`.md` extensions are preserved.

### Tab Bar (`src/components/Tabs/TabBar.svelte`)

**Container:** Full width of its editor pane. Height: 36px. Background `--color-surface`. Bottom border 1px `--color-border`. `display: flex; flex-direction: row; overflow-x: auto; scrollbar-width: none` (hidden horizontal scroll — tabs scroll with keyboard or mouse wheel, no visible scrollbar).

**Tab anatomy (`src/components/Tabs/Tab.svelte`):**
1. Padding: 8px horizontal, 8px vertical
2. Filename: Body (14px/400), `--color-text-muted` for inactive, `--color-text` weight 700 for active
3. Dirty dot: 6px × 6px filled circle `--color-accent`, shown when tab has unsaved content. (Note: auto-save fires every 2s so this rarely persists.)
4. Close button: lucide `X` icon (14px). Visible always on hover of the tab; hidden at rest for inactive tabs, always visible for active tab. 20px × 20px hit target. Hover color `--color-error`.
5. Max-width: 160px. Truncation: ellipsis at end. Tooltip with full filename.

**Active tab indicator:** 2px bottom border `--color-accent`. Background `--color-surface`. Inactive tabs: background `--color-bg`.

**Tab drag-to-reorder:** HTML5 drag-and-drop. Dragging a tab shows a ghost at 50% opacity. Drop zone highlights with `--color-accent` bottom border on the gap between tabs.

**Drag-to-split:** Dragging a tab to the left edge (< 40px from pane left) or right edge (> pane width - 40px) shows a full-height overlay indicator — a 4px `--color-accent` vertical line with `--color-accent-bg` fill at 20% opacity covering the half of the editor area where the new pane would appear. Drop creates the split.

### Editor Pane (`src/components/Editor/EditorPane.svelte`)

**Composition:** `TabBar` (36px fixed) + `CMEditor` (fills remaining height). Background `--color-surface`.

**Empty pane state (no tabs open):** Centered vertically and horizontally in the pane area. See copywriting contract — "No file open" state.

**Readonly overlay (ERR-03):** Full-pane overlay div with `pointer-events: none`, `background: rgba(255,255,255,0.6)`, `position: absolute; inset: 0; z-index: 10`. Does not obscure text — editor content remains readable. No readonly icon or badge in Phase 2.

### Confirmation Dialog (rename wiki-link count + delete)

**Not a modal overlay** — implemented as an inline popover anchored to the sidebar row that triggered it. Pattern: `position: fixed`, 280px wide, `--color-surface` background, 1px `--color-border` border, 8px border-radius, shadow `0 4px 16px rgba(0,0,0,0.14)`, 16px padding.

**Structure:**
1. Heading: Heading role (20px/700) — "Move to Trash?" or "Rename file?"
2. Body: Body (14px/400), `--color-text`, 8px bottom margin — see copywriting contract for exact strings
3. Button row: 8px gap, right-aligned
   - Cancel button: 14px/400, background transparent, border 1px `--color-border`, 8px × 16px padding, 6px border-radius, hover background `--color-bg`
   - Confirm button: 14px/400, white text
     - Delete: background `--color-error`, hover brightness 90%
     - Rename: background `--color-accent`, hover brightness 90%

**Focus management:** On open, focus jumps to Confirm button. Escape closes and returns focus to the triggering element.

### Progress UI (bulk watcher events — SYNC-05)

**Reuses Phase 1 `ProgressBar` component.** When bulk-change threshold is exceeded (D-13: >500 events / 2s), the progress bar re-appears at the top of the sidebar tree area (not a modal). Height: 40px strip replacing the tree header. Same anatomy as Phase 1: "Scanning changes…" label + counter + bar. Disappears when burst subsides.

No changes to the `ProgressBar.svelte` component itself — only the trigger condition changes from Phase 1 (vault open) to Phase 2 (bulk watcher event burst).

### Toast System

**Unchanged from Phase 1.** Phase 2 wires two new toast triggers using the existing `toastStore`:

- SYNC-08 clean-merge: `clean-merge` variant — "External changes merged into {filename}."
- SYNC-08 conflict: `conflict` variant — "Conflict in {filename} — local version kept."
- ERR-03 vault unmount: `error` variant — "Vault unavailable. Editing disabled."
- ERR-04 disk-full during auto-save: `error` variant — already wired in Phase 1's `write_file` path

## Performance Contracts

UI-level constraints the Phase 2 frontend must not violate:

| Constraint | Value | Source |
|------------|-------|--------|
| Keystroke latency | < 16 ms | PERF-04 — CMEditor must not trigger Svelte re-render per keystroke; `updateListener` only |
| Open note (click → editor ready) | < 100 ms | PERF-03 — `read_file` IPC + CM6 mount |
| Sidebar tree expand (click → subtree visible) | < 100 ms | PERF-03 analog — `list_directory` IPC + DOM update |
| Watcher event → toast appear | < 200 ms from Tauri event | Synchronous toastStore update |
| Sidebar resize drag | 60 fps | CSS variable update on `mousemove` — no JS layout reflow |
| Tab switch (click → editor swap) | < 50 ms | tabStore update + CMEditor {#key} remount |
| Bulk-change progress bar update | < 16 ms per frame | Same throttle pattern as Phase 1 (50ms minimum event interval) |

**Lazy-load constraint:** The sidebar tree must NOT recursively preload all subdirectories on vault open. Root-level entries only are fetched initially. `list_directory(path)` is called per-folder on first expand. This is the primary mechanism ensuring Phase 2 startup time remains within PERF-01 (< 3 s cold start) with a 100k-note vault.

**Tab remount strategy:** When switching tabs, CMEditor is remounted via `{#key activeTabId}`. This is the approved pattern from Phase 1 (RC-01) — EditorView is stored in `let` (not `$state`), so remount is clean. Scroll position and cursor position are saved to `tabStore` on tab deactivate and restored on mount.
