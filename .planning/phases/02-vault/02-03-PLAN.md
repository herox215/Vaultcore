1. **src/store/tabStore.ts** — New store following the classic `writable` pattern (D-06/RC-01):

   ```typescript
   import { writable } from "svelte/store";

   export interface Tab {
     id: string;
     filePath: string;
     isDirty: boolean;
     scrollPos: number;
     cursorPos: number;
     lastSaved: number;
   }

   export interface SplitState {
     left: string[];    // Tab IDs in left pane
     right: string[];   // Tab IDs in right pane (empty = no split)
     activePane: "left" | "right";
   }

   export interface TabStoreState {
     tabs: Tab[];
     activeTabId: string | null;
     splitState: SplitState;
   }
   ```

   Action methods on the store object:
   - `openTab(filePath: string): string` — If tab with same filePath exists, activate it and return its id. Otherwise create new Tab with `id: crypto.randomUUID()`, add to current pane's tab list in splitState, set as activeTabId. Return id.
   - `closeTab(tabId: string): void` — Remove from tabs array and from whichever pane's list it was in. If was active: activate left sibling (or right if leftmost). If pane becomes empty and split was active: merge remaining pane to left, set `splitState.right = []`, set `splitState.activePane = "left"`.
   - `activateTab(tabId: string): void` — Set activeTabId, set activePane to whichever pane contains it.
   - `cycleTab(direction: 1 | -1): void` — Within the active pane's tab list, move to next/prev tab (wrapping). `direction: 1` = forward (Cmd+Tab), `-1` = backward (Cmd+Shift+Tab).
   - `moveToPane(pane: "left" | "right"): void` — Move active tab to specified pane. If target pane was empty, this creates the split. If source pane becomes empty after move, close that pane.
   - `setDirty(tabId: string, dirty: boolean): void` — Update `isDirty` flag on the tab.
   - `updateScrollPos(tabId: string, scrollPos: number, cursorPos: number): void` — Save scroll/cursor position for restore on re-activate.
   - `getActiveTab(): Tab | null` — Derived helper (not a store method, just reads current state).
   - `updateFilePath(oldPath: string, newPath: string): void` — When a file is renamed/moved, update the matching tab's filePath.
   - `closeByPath(filePath: string): void` — Close any tab with matching filePath (used when file is deleted).
   - `_reset(): void` — Test helper, resets to initial state.

   Initial state:
   ```typescript
   const initial: TabStoreState = {
     tabs: [],
     activeTabId: null,
     splitState: { left: [], right: [], activePane: "left" },
   };
   ```

1. **src/components/Tabs/Tab.svelte** — Per UI-SPEC:
   - Props: `tab: Tab`, `isActive: boolean`.
   - Layout: 8px horizontal + 8px vertical padding. Filename 14px/400 `--color-text-muted` (inactive) or `--color-text` weight 700 (active). Max-width 160px, ellipsis truncation. Tooltip with full filename.
   - Dirty dot: 6px circle `--color-accent`, shown when `tab.isDirty`.
   - Close button: lucide `X` (14px), 20x20 hit target. Hidden at rest for inactive, always visible for active, visible on hover for all. Hover: icon color `--color-error`.
   - Active indicator: 2px bottom border `--color-accent`. Background `--color-surface` (active) or `--color-bg` (inactive).
   - Click: dispatch `activate` event.
   - Middle-click (`event.button === 1`): dispatch `close` event.
   - Close button click: dispatch `close` event, `event.stopPropagation()`.

   **Drag-to-reorder:**
   - `draggable="true"`.
   - On `dragstart`: set `event.dataTransfer.setData("text/vaultcore-tab", tab.id)`, `effectAllowed = "move"`.
   - Ghost at 50% opacity via `event.dataTransfer.setDragImage`.

   **Drag-to-split detection** is handled by EditorPane (not Tab).

2. **src/components/Tabs/TabBar.svelte** — Per UI-SPEC:
   - Props: `paneId: "left" | "right"`, `tabs: Tab[]`, `activeTabId: string | null`.
   - Container: full width, 36px height, background `--color-surface`, bottom border 1px `--color-border`. `display: flex; overflow-x: auto; scrollbar-width: none`.
   - Render `<Tab>` for each tab. Handle `activate` and `close` events by calling `tabStore.activateTab(id)` / `tabStore.closeTab(id)`.
   - Drag-to-reorder drop zone: on `dragover` check `text/vaultcore-tab` type, show insertion indicator (2px `--color-accent` vertical line between tabs). On `drop`, reorder tabs in the pane.
   - Keyboard: Cmd/Ctrl+Tab calls `tabStore.cycleTab(1)`. Cmd/Ctrl+Shift+Tab calls `tabStore.cycleTab(-1)`. Cmd/Ctrl+W calls `tabStore.closeTab(activeTabId)`. Register these as global keyboard listeners in VaultLayout (not per-TabBar) to avoid duplication.

3. **src/components/Editor/EditorPane.svelte** — Per UI-SPEC:
   - Props: `paneId: "left" | "right"`.
   - Composition: `<TabBar>` (36px fixed) + editor content area (fills remaining height).
   - Subscribe to `tabStore` for the pane's tab list and active tab.
   - Background `--color-surface`.

   **EditorView lifecycle (CRITICAL — Pitfall 4 from RESEARCH):**
   - Maintain a `Map<string, EditorView>` at the EditorPane level (module-level variable, NOT reactive state).
   - When a new tab is opened: read file content via `readFile(tab.filePath)`, create a new EditorView, store in the Map.
   - When switching tabs: save scroll/cursor position of current tab to tabStore (`updateScrollPos`), hide current EditorView container (`display: none`), show target EditorView container (`display: block`), restore scroll/cursor position.
   - When closing a tab: destroy the EditorView (`view.destroy()`), remove from Map.
   - This preserves undo history across tab switches (no `{#key}` remount on switch).
   - Each EditorView gets the same extensions as Phase 1 (import `buildExtensions` from extensions.ts, `autoSaveExtension` from autoSave.ts).
   - The auto-save extension writes via `writeFile` and updates `tabStore.setDirty(tabId, false)` on successful save, `tabStore.setDirty(tabId, true)` on doc change.

   **Empty pane state:** When pane has no tabs, show centered: heading "No file open" (14px `--color-text-muted`), body "Select a file from the sidebar, or drag a tab here to split the view." (14px `--color-text-muted`).

   **Readonly overlay (ERR-03 placeholder):** Subscribe to `vaultStore` for a future `vaultReachable` flag. When false, overlay with `pointer-events: none; background: rgba(255,255,255,0.6); position: absolute; inset: 0; z-index: 10`. Implementation in Plan 05 wires the actual trigger.

   **Drag-to-split detection:**
   - On `dragover` with `text/vaultcore-tab` type: if cursor is within 40px of the left or right edge of the pane, show split indicator (4px `--color-accent` vertical line + 20% opacity `--color-accent-bg` fill on the half where the new pane would appear).
   - On `drop` at edge: call `tabStore.moveToPane("right")` or `tabStore.moveToPane("left")` as appropriate.
