# Phase 5.1: Phase 5 UAT Bugfixes — Summary

## What was fixed

### Bug 1 — Settings modal too small (`06cc8ad`)
Modal had no `min-height`, so it collapsed to content size and felt cramped. Bumped to 600×560min, 85vh max, header 48px with 24px padding. All three sections now breathe.

### Bug 2 — Tag count stale + per-file semantics (`eaea7f7`)
**Architectural root cause:** `write_file` in Rust never dispatched `UpdateLinks`/`UpdateTags` to IndexCoordinator. `write_ignore` suppresses the watcher event for self-writes to avoid double-indexing, but nothing took its place, so in-memory TagIndex + LinkGraph were stale from the first auto-save until cold restart.

**Fix 1:** `write_file` now explicitly dispatches `UpdateLinks` + `UpdateTags` after a successful write. This also fixes a latent Phase 4 bug (LinkGraph also went stale after auto-save, silently affecting backlinks + unresolved-link colors).

**Fix 2:** Tag count changed from per-file-unique to total-occurrence. Writing `#test` 3× in one note now surfaces `test(3)` instead of `test(1)`. New helper `extract_inline_tag_occurrences` preserves duplicates; original `extract_inline_tags` with per-file dedup kept for other callers.

### Bug 3 — Tag click doesn't search (`937af6f`)
`TagsPanel.runSearchFor` set `activeTab` + `query` but had no consumer to trigger `search_fulltext`. User saw the Search tab with the query in the input but no results.

**Fix:** Added `searchStore.runSearch(query)` that atomically sets query + activeTab AND dispatches the IPC. `SearchInput` gets an `externalValue` prop so programmatic query updates reflect in the input box.

### Bug 4 — YAML frontmatter tags descoped (`eaea7f7`, same commit as Bug 2)
`TagIndex.update_file` no longer calls `extract_yaml_tags`. The function stays in the module as dead code for future re-enablement.

### Bug 5 — Cmd+\ sidebar toggle unusable on German keyboards (`06cc8ad`)
On German QWERTZ, `\` requires AltGr+ß so Ctrl+\ is awkward. Added Cmd/Ctrl+Shift+E as an ergonomic alias (VSCode Explorer-toggle convention). Both bindings appear in the Settings shortcut table. Spec UI-03 still satisfied.

### Bug 6 — Fenced code blocks rendered with no syntax colors (`b568fea`)
`markdownHighlightStyle` only defined tag mappings for Markdown structural tokens (heading, strong, link, etc.). The 20+ code-token tags that Lezer grammars emit (`keyword`, `string`, `number`, `function`, `className`, `typeName`, `operator`, etc.) had no mappings, so highlighted tokens rendered as plain text color.

**Fix:** Extended the HighlightStyle with 20+ code-token mappings. Keywords use accent color (dark-mode friendly), strings green, numbers orange, functions blue, types pink, operators muted. Added `.cm-FencedCode` block styling (background + mono font + padding + rounded corners).

### Bug 10 — Sort menu buggy → removed (`bc23c92`)
Removed the ArrowUpDown button + SortMenu component from Sidebar. `treeState` retains `sortBy` (defaults to `"name"`) for future re-enablement, but no UI exposes it.

### Bug 11 — Folders don't stay expanded on restart (`bc23c92`)
Sidebar's `onMount` read `$vaultStore.currentPath` synchronously, but App.svelte's auto-load-last-vault is async. On cold start, Sidebar mounted before the vault was opened, so `loadTreeState()` was skipped, leaving `treeState` empty. First render used empty `expanded` list → folders rendered collapsed.

**Fix:** Subscribe to `vaultStore` and re-run `loadTreeState` + `loadRoot` + `tagsStore.reload` whenever `currentPath` transitions to a new value. Mirrors `EditorPane.reloadResolvedLinks` pattern.
