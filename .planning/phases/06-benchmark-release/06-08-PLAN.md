# VaultCore

> **Stay fluid at 100,000+ notes.**
>
> A local, Markdown-first knowledge-management desktop app built on Tauri 2 + Rust + CodeMirror 6. No cloud, no telemetry, files never leave your disk.

## Downloads

Download the latest alpha: **[v0.1.0-alpha.1](https://github.com/<org>/<repo>/releases/tag/v0.1.0-alpha.1)**.

Platforms: macOS (Intel + Apple Silicon), Windows 10/11, Linux (Ubuntu 22.04+, Fedora 38+).

### Unsigned alpha caveats

- **macOS:** first launch is blocked by Gatekeeper. In Terminal: `xattr -d com.apple.quarantine /Applications/VaultCore.app`, then open.
- **Windows:** SmartScreen shows "Publisher unknown". Click "More info" → "Run anyway".
- **Linux (AppImage):** `chmod +x VaultCore*.AppImage && ./VaultCore*.AppImage`.

Signing + notarisation land in **v0.1.0-beta**.

## Core value

VaultCore is built for power users whose vaults have grown past the point where Obsidian starts to lag (≈100,000+ notes). It keeps every file as plain Markdown on disk — opens existing Obsidian vaults without corruption — and hits every spec §7 performance budget on a generated 100k-note vault (see [`PERF-REPORT-FILLED.md`](.planning/phases/06-benchmark-release/artifacts/PERF-REPORT-FILLED.md)).

## Security

- Zero network calls, verified at runtime (`strace -e trace=%network`)
- Zero telemetry, zero analytics
- Vault data never leaves the local filesystem
