```rust
use pulldown_cmark::{Parser, Options, Event};

pub fn strip_markdown(md: &str) -> String {
    let parser = Parser::new_ext(md, Options::empty());
    let mut plain = String::with_capacity(md.len());
    for event in parser {
        match event {
            Event::Text(t) | Event::Code(t) => plain.push_str(&t),
            Event::SoftBreak | Event::HardBreak => plain.push(' '),
            _ => {}
        }
    }
    plain
}
```

**3. src-tauri/src/indexer/memory.rs** — In-memory file metadata store:
```rust
use std::collections::HashMap;
use std::path::PathBuf;

#[derive(Debug, Clone)]
pub struct FileMeta {
    pub relative_path: String,  // forward-slash normalized, relative to vault root
    pub hash: String,           // SHA-256 hex
    pub title: String,          // first H1 or filename stem
}

#[derive(Debug, Default)]
pub struct FileIndex {
    entries: HashMap<PathBuf, FileMeta>,  // keyed by canonical absolute path
}

impl FileIndex {
    pub fn new() -> Self { Self::default() }
    pub fn insert(&mut self, abs_path: PathBuf, meta: FileMeta) { self.entries.insert(abs_path, meta); }
    pub fn get(&self, abs_path: &PathBuf) -> Option<&FileMeta> { self.entries.get(abs_path) }
    pub fn remove(&mut self, abs_path: &PathBuf) -> Option<FileMeta> { self.entries.remove(abs_path) }
    pub fn clear(&mut self) { self.entries.clear(); }
    pub fn len(&self) -> usize { self.entries.len() }
    pub fn is_empty(&self) -> bool { self.entries.is_empty() }
    pub fn all_relative_paths(&self) -> Vec<String> {
        self.entries.values().map(|m| m.relative_path.clone()).collect()
    }
    pub fn all_entries(&self) -> impl Iterator<Item = (&PathBuf, &FileMeta)> { self.entries.iter() }
}
```

**1. Complete IndexCoordinator::index_vault method** in `src-tauri/src/indexer/mod.rs`:

```rust
pub async fn index_vault(&self, vault_path: &Path, app: &AppHandle) -> Result<VaultInfo, VaultError>
```

Flow:
1. Compute `vaultcore_dir = vault_path.join(".vaultcore")` and `index_dir = vaultcore_dir.join("index").join("tantivy")`.
2. Call `tantivy::check_version(&vaultcore_dir)`. If `false`, delete `index_dir` if it exists, log "Schema mismatch — rebuilding index", emit toast event `vault://index_toast` with `{ message: "Index wird neu aufgebaut...", variant: "clean-merge" }`.
3. Call `tantivy::open_or_create_index(&index_dir, &schema)`. On error, map to `VaultError::IndexCorrupt`, delete `index_dir`, retry once (ERR-02 auto-rebuild). If retry fails, return `IndexCorrupt`.
4. Walk vault with `WalkDir::new(vault_path)` filtering: skip `.obsidian/`, skip `.vaultcore/`, skip `.trash/`, only `.md` extension, skip non-UTF-8 via `read_to_string` match (IDX-08).
5. Collect all `.md` paths into a `Vec<PathBuf>`. Set `total = paths.len()`.
6. For each path:
   - `read_to_string`. On `Err` (non-UTF-8): `continue` silently (IDX-08).
   - Compute `hash = hash_bytes(content.as_bytes())`.
   - Check `file_index.get(&abs_path)`: if hash matches, `continue` (IDX-03 incremental skip).
   - Strip markdown: `body = parser::strip_markdown(&content)`.
   - Extract title: `title = tantivy::extract_title(&content, stem)`.
   - Compute `relative_path` with forward-slash normalization (existing convention from Phase 1).
   - Send `IndexCmd::AddFile { path, title, body, hash }` via `self.tx.send(...).await`.
   - Update `file_index` with new `FileMeta`.
   - Emit `vault://index_progress` with `IndexProgressPayload { current, total, current_file }` throttled at 50ms (reuse existing PROGRESS_THROTTLE pattern).
7. Send `IndexCmd::Commit`.
8. Call `tantivy::write_version(&vaultcore_dir)`.
9. Build and return `VaultInfo { path, file_count, file_list }`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vault files -> Indexer | Untrusted Markdown content from vault files enters the indexer and Tantivy |
| Tantivy index -> Search results | Index data rendered as snippets in the UI via {@html} |
| Filesystem -> index_version.json | Sidecar JSON file could be manually tampered |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | indexer/parser.rs | mitigate | strip_markdown uses pulldown-cmark to emit plain text only — no HTML tags pass through to the Tantivy body field. This prevents XSS when SnippetGenerator wraps matches in `<b>` tags (Pitfall 5 from RESEARCH.md) |
| T-03-02 | Denial of Service | indexer/mod.rs | mitigate | mpsc channel depth capped at 1024; try_send for watcher events drops on full channel instead of blocking the watcher thread |
| T-03-03 | Tampering | indexer/tantivy.rs | accept | index_version.json can be manually edited by the user; worst case is an unnecessary rebuild — no security impact since the index is a rebuildable cache |
| T-03-04 | Information Disclosure | .vaultcore/index/ | mitigate | Index directory stored inside vault at .vaultcore/ (dot-prefix hidden); contains only search index data derived from files the user already owns |
| T-03-05 | Elevation of Privilege | commands/vault.rs | mitigate | IndexCoordinator receives only canonicalized paths that passed the existing vault-scope guard (T-02 from Phase 1); no new path input vectors |
</threat_model>
