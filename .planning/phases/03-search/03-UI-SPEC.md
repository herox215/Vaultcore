## Interaction Contracts

### Sidebar Tab Switching (D-01)

- Two tabs rendered at top of sidebar: "Dateien" (file tree) and "Suche" (search).
- Active tab: 12px / 600 weight, `--color-accent` bottom border 2px, background `--color-accent-bg` tint.
- Inactive tab: 12px / 400 weight, `--color-text-muted`, no bottom border.
- Keyboard: `Cmd/Ctrl+Shift+F` directly activates the Suche tab AND focuses the search input, even if Files tab is active.
- Tab switch: instant — no transition, no layout shift. File tree and search panel share the same container; only `display` toggled.
- Tab bar height: 32px. Border-bottom: 1px solid `--color-border`.

### Search Input (D-03)

- Placeholder text: "Suchen... (AND, OR, NOT, \"phrase\")" — gives syntax hint inline.
- Debounce: 200ms after last keystroke before invoking `search_fulltext`.
- Clear button (×): appears when input is non-empty. 16px lucide-svelte X icon. Clears input and resets results.
- Focus ring: `outline: 2px solid var(--color-accent)`, `outline-offset: 2px`, `border-radius: 4px`.
- During index rebuild: input disabled, `cursor: not-allowed`, background `--color-border`.

### Search Results (D-02, D-08)

- Counter row at top: "23 Treffer in 12 Dateien" — 12px / 400 / `--color-text-muted`. Appears as soon as results arrive.
- Overflow cap: if >100 results, show below list: "Zeige 100 von 342 Treffern — Suche verfeinern" (12px / 400 / muted, italic).
- Each result row: filename on first line (14px/700), snippet on second line (12px/400, muted). Matched term in snippet: `<mark>` element styled with `background: #FEF9C3, border-radius: 2px, padding: 0 2px`.
- Row hover: background `--color-accent-bg`.
- Row height: variable (2-line content) — minimum 48px, padding 8px 12px.
- Clicking a result: opens file (new tab or switch to existing), scrolls to match, triggers flash highlight.

### Flash Highlight on Scroll-to-Match (D-07)

- CM6 decoration applied to matched text range.
- Background: `#FEF9C3` (yellow-100) at opacity 1, transitions to opacity 0 over 2.5 seconds.
- CSS transition: `background-color 2500ms ease-out, opacity 2500ms ease-out`.
- After fade: decoration removed from CM6 state (no lingering mark).
- Only one flash highlight active at a time. New result click clears previous before applying.

### Quick Switcher (D-04, D-05, D-06)

- Trigger: `Cmd/Ctrl+P`. Dismiss: `Escape` or click outside modal.
- Mount point: `<body>` portal (avoids stacking context issues with sidebar layout).
- Backdrop: `position: fixed; inset: 0; background: rgba(0,0,0,0.4); z-index: 200`.
- Modal card: `width: 560px; max-height: 480px; position: fixed; top: 15%; left: 50%; transform: translateX(-50%); background: --color-surface; border: 1px solid --color-border; border-radius: 8px; box-shadow: 0 8px 32px rgba(0,0,0,0.2)`.
- Search input at top: full-width, 36px height, 16px horizontal padding, `border-bottom: 1px solid --color-border` (no outer border on input itself — card border frames it).
- Result list: scrollable, max-height computed from modal max-height minus input height.
- Result rows: filename (14px) left-aligned. Matched chars: 700 weight + `--color-accent` color. Relative path: 12px / `--color-text-muted`, right-aligned or below filename.
- Keyboard navigation: `ArrowUp`/`ArrowDown` move selection. Selected row: `background: --color-accent-bg`. `Enter` opens selected. Mouse hover also sets selection.
- Empty (no input): shows recently opened files from tabStore (label: "Zuletzt geöffnet"). Max 8 recents shown.
- No results found: single row "Keine Dateien gefunden" (12px / muted, centered).
- Fuzzy match algorithm: substring + word-initial. "mn" matches "meeting-notes.md". No regex required of user.

### Index Rebuild (D-09, D-10, D-11)

- Rebuild button in search panel header: "Index neu aufbauen" label + lucide-svelte `RefreshCw` icon (16px, strokeWidth 1.5). Right-aligned in header.
- During rebuild: button shows `RefreshCw` with CSS `animation: spin 1s linear infinite`. Label unchanged.
- Rebuild blocks search panel interaction. Editor and file tree remain fully interactive.
- Toast on start: `{ variant: "clean-merge", message: "Index wird neu aufgebaut..." }` — no auto-dismiss (persists until done).
- Toast on complete: `{ variant: "clean-merge", message: "Index aktualisiert" }` — auto-dismisses after 5s (existing toast behavior).
- Toast on IndexCorrupt auto-rebuild: `{ variant: "error", message: "Index beschädigt — wird neu aufgebaut..." }`.
- ProgressBar overlay: same existing component, same `vault://index_progress` event channel.

## Accessibility Contract

- Search input: `role="searchbox"`, `aria-label="Volltextsuche"`, `aria-busy="true"` during rebuild.
- Results list: `role="listbox"`, each row `role="option"`, `aria-selected` tracks keyboard selection.
- Quick Switcher modal: `role="dialog"`, `aria-modal="true"`, `aria-label="Schnellwechsler"`. Focus trapped inside while open.
- Rebuild button: `aria-label="Index neu aufbauen"`, `aria-disabled="true"` during active rebuild.
- Flash highlight: purely visual — no ARIA announcement needed (screen reader hears file open, which is announcement enough).
- Tab bar: `role="tablist"`, each tab `role="tab"`, `aria-selected`. Panel `role="tabpanel"`.
- Progress bar: existing `role="progressbar"` with `aria-valuemin/max/now` — unchanged.
