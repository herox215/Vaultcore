### Dark Mode (new — D-05, D-06, Claude's Discretion)

Applied under `:root[data-theme=dark]` in `tailwind.css`. Auto-mode uses `@media (prefers-color-scheme: dark)` within `:root[data-theme=auto]`. All 10 variables are re-declared; no new variable names introduced.

| CSS Variable | Dark Hex | Derivation rationale |
|---|---|---|
| `--color-bg` | `#1C1C1A` | Warm near-black — preserves the warm neutral spirit of light mode (D-06 spec verbatim) |
| `--color-surface` | `#2A2A28` | One step lighter than bg — warm dark surface for cards, modals |
| `--color-border` | `#3A3A38` | Subtle warm grey border — sufficient contrast against surface without glowing |
| `--color-text` | `#F5F5F4` | Warm off-white — direct inversion of light mode `--color-text` `#1C1C1A` |
| `--color-text-muted` | `#A8A8A5` | Warm light grey — legible at 4.5:1 against `#2A2A28` surface |
| `--color-accent` | `#7C3AED` | Violet one step lighter than `#6D28D9` — WCAG AA against dark bg (contrast ~5.2:1 on `#1C1C1A`) |
| `--color-accent-bg` | `#2E1F4A` | Deep violet overlay — replaces `#EDE9FE` in dark context; warm undertone |
| `--color-error` | `#F87171` | Red 400 — softer than `#DC2626` on dark bg; WCAG AA against `#1C1C1A` |
| `--color-warning` | `#FCD34D` | Amber 300 — legible on dark without burning; replaces `#D97706` |
| `--color-success` | `#34D399` | Emerald 400 — warm-toned success; replaces `#059669` |
| `--color-code-bg` | `#252523` | Just above bg — creates code block visual separation without jarring contrast |

WCAG notes:
- `--color-text` `#F5F5F4` on `--color-bg` `#1C1C1A`: contrast ~18:1 — AAA.
- `--color-text-muted` `#A8A8A5` on `--color-surface` `#2A2A28`: contrast ~5.1:1 — AA.
- `--color-accent` `#7C3AED` on `--color-bg` `#1C1C1A`: contrast ~5.2:1 — AA (large text and UI components).

## New UI Surfaces — Visual Specifications

### 1. Tag Panel (TAG-01..04, D-03, D-04)

Location: Third tab in the existing sidebar tab strip. Tab label: "Tags". Tab icon: lucide `Hash` (16×16px) placed left of the label text.

Tab strip after Phase 5: `Dateien | Suche | Tags` — each tab flex: 1 (equal widths, inherits `.vc-sidebar-tab` styles).

**Tag panel layout:**
- Outer container: `flex flex-col h-full overflow-hidden`.
- Inner scroll area: `flex-1 overflow-y-auto` — same pattern as file tree and search results.
- Top of panel: no separate toolbar. Sort is handled at sidebar topbar level (see Sort Menu below).
- Empty state: centered in scroll area, vertically aligned to 40% from top.

**Tag row (flat / top-level tag):**
- Height: 28px minimum.
- Padding: 0px vertical (row fills from line-height), 16px horizontal.
- Layout: `flex items-center justify-between`.
- Left: `#tag-name` text at 14px/400/`--color-text`.
- Right: usage count badge — `(12)` at 12px/400/`--color-text-muted`. No background pill for MVP.
- Hover: `--color-accent-bg` background on full row width.
- Cursor: pointer.
- No focus ring on tag rows in Phase 5 (keyboard nav deferred — tag panel is mouse-primary for MVP).

**Nested tag row (`#parent/child`):**
- Displayed as a tree: parent row first, then child rows indented 16px from parent left edge.
- Expand/collapse toggle: lucide `ChevronRight` (12×12px) rotated 90° when expanded, placed at left of parent row before the `#` label. Toggle area: 16px wide, full row height.
- Child rows inherit same row anatomy. No further nesting levels in MVP (spec only mentions one level of nesting — `#parent/child`).

**Tag-click behavior (TAG-04, D-04):**
- Click on any tag row → switches sidebar to "Suche" tab, prefills search input with `#tag-name`, fires `search_fulltext` IPC.
- No new tab opened; search results appear in the Search panel (existing SearchPanel component).
- No visual feedback beyond the tab switch and search results populating.

## Interaction Contracts

### Theme Toggle (UI-01, D-05, D-07)

- Settings modal → "Erscheinungsbild" radio group → selecting a value:
  1. Updates `themeStore` writable (sets `'light' | 'dark' | 'auto'`).
  2. Store subscriber sets `document.documentElement.dataset.theme` to the chosen value.
  3. Persists to `localStorage['vaultcore-theme']`.
  4. CSS variable cascade takes effect immediately — no page reload, no editor rebuild.
- On app start: `themeStore` reads `localStorage['vaultcore-theme']` and applies. Default: `'auto'`.
- `auto` mode: `:root[data-theme=auto]` contains `@media (prefers-color-scheme: dark)` block re-declaring dark variables. OS-level switch is instant.

### Font / Size Settings (UI-02, D-08, D-09)

- Body font dropdown change:
  1. Updates `settingsStore.fontBody`.
  2. Sets `document.documentElement.style.setProperty('--vc-font-body', fontStack)`.
  3. Persists to `localStorage['vaultcore-font-body']`.
- Mono font dropdown change: same pattern via `--vc-font-mono` / `localStorage['vaultcore-font-mono']`.
- Size slider change:
  1. Updates `settingsStore.fontSize`.
  2. Sets `--vc-font-size` on `<html>` to `{value}px`.
  3. Persists to `localStorage['vaultcore-font-size']`.
- Changes are live (no "Apply" button needed — slider and dropdown update reactively).
- Font options: if a `@fontsource/*` package is not installed, its dropdown option renders with `disabled` attribute and `--color-text-muted` text — no toast, no crash.

### Sidebar Collapse (UI-03, D-12)

- `Cmd/Ctrl+\` → toggles sidebar width between current width and 0px.
- Registered in `SHORTCUTS` array with id `toggle-sidebar`.
- Transition: `width 200ms ease` (matches right sidebar animation from Phase 4).
- When collapsed: sidebar tab strip is also hidden (sidebar is zero-width, overflow hidden).
- State not persisted (sidebar collapse is session-only per spec — no localStorage entry for this).

### New Note (EDIT-11, D-12)

- `Cmd/Ctrl+N` → registered in `SHORTCUTS` array with id `new-note`.
- Behavior: calls `create_file` IPC with path `{currently-selected-folder or vault root}/Unbenannte Notiz.md`.
- If `Unbenannte Notiz.md` already exists: appends ` 2`, ` 3`, etc. (backend handles uniqueness).
- Opens the new file in a new tab via `tabStore.openTab`.
- After opening: triggers `InlineRename` focus on the new file's TreeNode so user can immediately type the filename.
- No toast on success. Error toast on IPC failure: "Neue Notiz konnte nicht erstellt werden."

### File Browser Sort (FILE-06, D-19, D-20)

- Sort button click → opens `SortMenu` popover.
- Selecting a sort option:
  1. Updates `treeStore` sort state.
  2. Re-sorts the currently-loaded file tree (client-side re-sort — no IPC needed if tree nodes already carry `modified` and `created` timestamps, which `list_directory` returns).
  3. Persists to `localStorage['vaultcore-tree-state:{vaultHash}']` as `{ sortBy: 'name'|'modified'|'created', expanded: string[] }`.
- Sort rules: folders always before files. Within category: name ascending, modified/created descending.
- Sort applies immediately — no animation.
- Active option highlighted in menu on next open.

### Folder Expand/Collapse Persistence (FILE-07, D-19)

- On any expand or collapse of a folder TreeNode:
  1. Updates the `expanded: string[]` set in `localStorage['vaultcore-tree-state:{vaultHash}']`.
  2. No debounce — immediate write (single JSON key, negligible I/O).
- On vault open: restored from localStorage before first render.
- Vault hash: `sha256(vault_path)` truncated to first 16 hex chars — used as a stable per-vault namespace key.

### Hash-Verify Auto-Save (EDIT-10, D-14, D-15)

- Before each auto-save write (2s idle debounce fires):
  1. Read current on-disk file hash via existing `write_file` return value pattern or a dedicated hash IPC.
  2. Compare against `tabStore.originalHash` for the active tab (already tracked from last successful read/write).
  3. **Match** → proceed with write as normal. Update `originalHash` to hash of newly written content.
  4. **Mismatch** → cancel write. Invoke Phase 2 three-way merge engine: `{local buffer content, on-disk content, last-known content at originalHash}`.
  5. Merged result flows back into CM6 doc via existing `applyMerge` path. Toasts: "Externe Änderungen wurden eingebunden" (clean merge) or "Konflikt in {file} – lokale Version behalten" (conflict).
- No new toast variants — reuses Phase 2 merge toast strings exactly.
- Performance: one hash read per 2s auto-save trigger. Hash computation is SHA-256 of file bytes — negligible CPU per PERF budget analysis (CONTEXT.md D-15).

### Settings Modal Close

- Click outside backdrop → close.
- Escape key → close (registered in modal's `onkeydown`, not in global `SHORTCUTS` — prevents interference with editor Escape handling).
- Click "X" button → close.
- Settings changes are applied live; there is no "Save" or "Cancel" button in the modal. Closing is always safe.

### Keyboard Shortcut Priority (Claude's Discretion)

Priority order when events arrive at `VaultLayout.handleKeydown`:

1. If Settings modal is open: only Escape is handled (close modal). All other shortcuts suppressed.
2. If Quick Switcher modal is open: only Escape and ArrowUp/Down/Enter pass to the switcher. All others suppressed.
3. If any inline rename input is focused: no global shortcuts fire.
4. Otherwise: `SHORTCUTS` array is iterated in declaration order. First match wins.

Shortcut declaration order in `SHORTCUTS`:
1. `Cmd+N` — new note
2. `Cmd+P` — quick switcher
3. `Cmd+Shift+F` — search
4. `Cmd+Shift+B` — backlinks toggle
5. `Cmd+\` — sidebar toggle
6. `Cmd+Tab` — next tab
7. `Cmd+W` — close tab

## Accessibility Contract

- Settings modal: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to the "Einstellungen" heading. Focus trapped within modal while open.
- Settings modal opens with focus on the first interactive element (theme radio group first option).
- Theme radio group: `role="radiogroup"`, each option `role="radio"`, `aria-checked`.
- Font dropdowns: native `<select>` — full keyboard and screen reader support inherited.
- Font size slider: native `<input type="range">` with `aria-label="Schriftgröße"`, `aria-valuemin="12"`, `aria-valuemax="20"`, `aria-valuenow="{current}"`.
- Shortcuts table: `role="table"`, `role="row"`, `role="cell"` — or a `<table>` element directly for native accessibility.
- `<kbd>` badges: rendered as `<kbd>` HTML elements — screen readers announce them as keyboard input.
- Sort menu popover: `role="menu"`, each item `role="menuitemradio"`, `aria-checked` on active. Dismiss on Escape.
- Tags panel: `role="tree"` on the list container, `role="treeitem"` on each tag row. Expanded state: `aria-expanded` on parent rows.
- Sort button: `aria-label="Sortierung"`, `aria-haspopup="menu"`, `aria-expanded` toggled on open/close.
- Settings gear button: `aria-label="Einstellungen"`, `aria-haspopup="dialog"`.
- Tag rows: `role="button"` or `<button>` element. `aria-label="{tag-name} — {N} Notizen"`.
- Keyboard navigation in tag panel: Tab to enter panel, ArrowUp/Down to navigate tag rows, ArrowRight to expand parent, ArrowLeft to collapse, Enter to trigger search.
