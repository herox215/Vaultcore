<objective>
Create CM6 ViewPlugin that decorates `[[wiki-links]]` in the editor with accent/muted colors and dispatches click events for the Svelte layer to handle. Expose a synchronous resolution API (`setResolvedLinks(map)` + `resolveTarget(stem)`) that EditorPane populates after vault open via the `get_resolved_links` IPC.

Purpose: This is the core editor-side feature users interact with — seeing their links styled and being able to click them. It transforms `[[text]]` from plain text into a navigable knowledge graph UI. The resolution API is synchronous by design: a single IPC populates the map on vault open, and every subsequent decoration / click lookup is an in-memory `Map.get(stem.toLowerCase())` — no IPC inside the CM6 render loop, no IPC at click time, and subfolder-correct navigation without a flat-vault stub.

Output: `wikiLink.ts` with ViewPlugin + click handler + `setResolvedLinks(map)` + `resolveTarget(stem)`, extensions.ts updated to include it.

       const WIKI_LINK_RE = /\[\[([^\]|]+?)(?:\|([^\]]*))?\]\]/g;

    3. Define the module-level resolution map and its setter/getter:
       ```typescript
       // Stem (lowercased) -> vault-relative path. Populated once per vault open
       // via EditorPane calling setResolvedLinks(map) after get_resolved_links IPC.
       // Incrementally updated on file create/delete/rename by EditorPane.
       let resolvedLinks: Map<string, string> = new Map();

       export function setResolvedLinks(map: Map<string, string>): void {
         resolvedLinks = map;
       }

       /**
        * Synchronous lookup used by both the ViewPlugin (decoration) and the
        * EditorPane click handler. Input is the raw target from `[[target]]`
        * (possibly with `.md` suffix or alias already stripped by caller).
        * Returns the vault-relative path, or null if unresolved.
        */
       export function resolveTarget(target: string): string | null {
         const stem = (target.endsWith(".md") ? target.slice(0, -3) : target).toLowerCase();
         return resolvedLinks.get(stem) ?? null;
       }
       ```

       This is the API contract consumed by 04-04: EditorPane calls `setResolvedLinks(await getResolvedLinks())` on vault open, and the click handler calls `resolveTarget(detail.target)` to get the rel_path — zero IPC at click time, subfolder-correct via the shortest-path resolution computed in Rust.

    4. Helper `isInsideCodeBlock(state: EditorState, pos: number): boolean`:
       ```typescript
       const node = syntaxTree(state).resolve(pos, 1);
       let cur = node;
       while (cur) {
         const name = cur.type.name;
         if (name === "FencedCode" || name === "CodeBlock" || name === "InlineCode") return true;
         cur = cur.parent;
       }
       return false;
       ```
       Note: lezer node names must be verified at runtime. If "FencedCode" is not correct, try logging `node.type.name` during development and adjust. The research assumption A1 flags this.

    5. Define `buildDecorations(view: EditorView): DecorationSet`:
       - Create `RangeSetBuilder<Decoration>`.
       - Get the document text: `view.state.doc.toString()`.
       - Iterate regex matches with `WIKI_LINK_RE.exec(text)`.
       - For each match:
         a. Check `isInsideCodeBlock(view.state, match.index)` — skip if true.
         b. Extract target (group 1), strip `.md` suffix.
         c. Determine if resolved: `const resolved = resolvedLinks.has(target.toLowerCase());`.
         d. Apply `Decoration.mark({ class: resolved ? "cm-wikilink-resolved" : "cm-wikilink-unresolved", attributes: { "data-wiki-target": target, "data-wiki-resolved": resolved ? "true" : "false" } })`.
         e. Add to builder: `builder.add(match.index, match.index + match[0].length, decoration)`.
       - CRITICAL: Collect all matches into array first, sort by `from` position, then add to RangeSetBuilder in order. RangeSetBuilder panics if ranges are out of order.
       - Reset regex lastIndex to 0 before use (global regex stateful).
       - Return `builder.finish()`.

    6. Define `wikiLinkPlugin` as ViewPlugin:
       ```typescript
       export const wikiLinkPlugin = ViewPlugin.fromClass(
         class {
           decorations: DecorationSet;
           constructor(view: EditorView) {
             this.decorations = buildDecorations(view);
           }
           update(update: ViewUpdate) {
             if (update.docChanged || update.viewportChanged) {
               this.decorations = buildDecorations(update.view);
             }
           }
         },
         {
           decorations: (v) => v.decorations,
           eventHandlers: {
             mousedown(event: MouseEvent, view: EditorView) {
               // Only handle Ctrl/Cmd+Click or plain click on decorated spans
               const target = event.target as HTMLElement;
               const wikiTarget = target.closest("[data-wiki-target]");
               if (!wikiTarget) return false;

               const linkTarget = wikiTarget.getAttribute("data-wiki-target");
               const isResolved = wikiTarget.getAttribute("data-wiki-resolved") === "true";
               if (!linkTarget) return false;

               event.preventDefault();
               event.stopPropagation();

               // Dispatch custom event for the Svelte layer to handle
               // (avoids importing store directly into CM6 extension).
               // The listener will call resolveTarget(linkTarget) for resolved
               // links to get the vault-relative path — NOT a flat-vault stub.
               view.dom.dispatchEvent(
                 new CustomEvent("wiki-link-click", {
                   bubbles: true,
                   detail: { target: linkTarget, resolved: isResolved },
                 })
               );
               return true;
             },
           },
         }
       );
       ```
