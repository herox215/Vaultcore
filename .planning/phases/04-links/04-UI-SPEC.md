## Interaction Contracts

### Resolved Wiki-Link Click (LINK-03)
- Click on `[[Note]]` in editor → opens target in new tab via `tabStore.openTab(absPath)`.
- Uses `scrollStore` one-shot pattern to scroll CM6 to top of new note (existing pattern from Phase 3).
- No hover underline — accent color alone signals clickability (keeps editor clean).
- Cursor: `pointer` on hover over decorated link spans.

### Unresolved Wiki-Link Click (D-08, LINK-01)
- Click on unresolved `[[Note]]` → calls `create_file` IPC → opens new empty note in new tab.
- No confirmation prompt — Zettelkasten workflow expects instant create.
- New note filename: the link target as written (stripped of alias suffix if `[[Note|alias]]`).
- Toast: none for successful create. Error toast on create failure (existing `toastStore.push` pattern).

### Autocomplete Popup (LINK-05, D-04, D-05, D-06)
- Triggered: user types `[[` in CM6 editor.
- Popup position: immediately below the cursor, aligned to the `[[` token start. Offset 4px below baseline.
- Popup width: 360px fixed. Max height: 320px with internal scroll.
- Row anatomy: top line — filename at 14px/600 in `--color-text`; bottom line — relative path at 12px/400 in `--color-text-muted`.
- Row height: auto, minimum 44px (two lines + 8px padding each side = ~52px typical).
- Active row: `--color-accent-bg` background, filename text shifts to `--color-accent`.
- Keyboard: ArrowDown/Up to navigate, Enter to select, Escape to dismiss, Tab closes and inserts `[[selectedname]]`.
- After selecting a file: popup closes, `[[Filename]]` inserted. If user then types `|`, alias freetext entry begins — popup does NOT reopen (D-06). Alias captured until `]]` or Enter.
- Empty state: "Keine Dateien gefunden" at 12px muted, centered, 16px padding.
- Filter: fuzzy via nucleo (same as Quick Switcher) — no minimum character count.
- Performance: call `search_filename` IPC on each keystroke after `[[`; budget < 10 ms (PERF-08).

### Backlinks Panel (LINK-06, D-01, D-02, D-03)
- Toggle: Cmd/Ctrl+Shift+B. State (open/closed) persisted to localStorage key `vaultcore-backlinks-open`.
- Default: closed.
- Width: 240px default, draggable between 180px–400px. Persisted to localStorage key `vaultcore-backlinks-width`.
- Header: "Backlinks" label at 12px/600, `--color-text-muted`. Right-aligned X close button (24×24px, lucide `X` icon).
- Empty state: "Kein Backlink für diese Notiz" at 14px/400 muted, vertically centered in panel.
- Backlink row: filename in 14px/600 `--color-text`, context snippet in 14px/400 `--color-text`, max 2 lines, `text-overflow: ellipsis` at line 2. Row padding: 8px vertical, 16px horizontal. Hover: `--color-accent-bg` background.
- Click on row: opens that note in a new tab (tabStore). Does NOT scroll to the specific link position (deferred to Phase 5).
- Separator: 1px `--color-border` between rows.
- Context snippet: 1–2 lines surrounding the `[[link]]`, with the link text itself rendered at `--color-accent` (small inline decoration matching editor style).
- No section grouping in Phase 4 — flat list ordered by filename ascending.
- Performance: `get_backlinks` IPC result arrives in < 20 ms (PERF-07); panel renders immediately on receive.

### Right Sidebar Layout Extension
- Grid: `VaultLayout` extends from `[sidebar][divider][1fr]` to `[sidebar][divider][1fr][right-divider][right-sidebar]`.
- Right divider: 4px, `--color-border`, `cursor: col-resize`. Hover/active: `--color-accent-bg`. Same pattern as left divider.
- Right sidebar column hidden (0px) when closed. Transition: `width 200ms ease` matching left sidebar.
- No animation beyond width transition — no slide-in, no fade.

### Rename-Cascade Confirmation (LINK-09, D-09, D-10)
- Trigger: rename or move of any file in sidebar (including drag-and-drop).
- Dialog replaces existing stub in `TreeNode.svelte`. Reuses the existing `showDeleteConfirm` inline pattern — no new modal component.
- Copy: **"X Links in Y Dateien werden aktualisiert. Fortfahren?"**
  - Where X = total link count, Y = affected file count (from `count_wiki_links` IPC).
  - If X = 0: no confirmation shown, rename proceeds silently (no prompt for zero-link renames).
- Buttons: [Abbrechen] (left, default neutral style) and [Aktualisieren] (right, `--color-accent` border and text).
- On confirm: `update_links_after_rename` IPC call. On partial failure: toast "X von Y Links aktualisiert. Z Dateien konnten nicht geändert werden." (D-10).
- On cancel: rename does NOT proceed (cancel = abort).
- Button width: auto, minimum 80px, 8px horizontal padding.
- Dialog display: inline below the TreeNode row (existing pattern), not a floating modal.

## Accessibility Contract

- All clickable links in CM6 editor: `cursor: pointer`, `role="link"` on the decorated span via CM6 widget.
- Autocomplete popup: `role="listbox"` on container, `role="option"` on each row, `aria-selected` on active row. Follows pattern from `QuickSwitcher.svelte`.
- Backlinks panel: `role="complementary"`, `aria-label="Backlinks"`. Row list: `role="list"` / `role="listitem"`.
- Right sidebar resize divider: `role="separator"`, `aria-orientation="vertical"`, `aria-label="Backlinks-Panel vergrößern"`.
- Rename dialog buttons: standard `<button>` elements, no ARIA augmentation needed.
- Keyboard navigation in autocomplete: ArrowUp/Down/Enter/Escape — no mouse required.
- Keyboard navigation in backlinks: Tab to enter panel, ArrowUp/Down to navigate rows, Enter to open note.
