<domain>
## Phase Boundary

Fix 5 bugs discovered during manual verification of Phase 4 (Links). All bugs are already root-cause-analyzed. No new capabilities, only repairs to already-shipped behavior.

**Bugs in scope:**
1. Unresolved-link click-to-create doesn't refresh sidebar tree
2. Autocomplete popup selected-row contrast (black-on-blue, unreadable detail)
3. Backlinks panel has no visible toggle — keyboard-only discovery
4. Rename-cascade silently fails because `incoming_for(old_path)` returns empty (race with watcher-driven LinkGraph update)
5. `closeBrackets()` extension auto-inserts `]]` when `[[` is typed, killing the autocomplete trigger

**Out of scope:** New features, refactoring unrelated code, additional UI polish.

</domain>

<decisions>
## Implementation Decisions

### Bug 1: Tree refresh after click-to-create
- **D-01:** Introduce a minimal **`treeRefreshStore`** (one-shot signal, pattern from `scrollStore`). `EditorPane.handleWikiLinkClick` bumps it after `createFile` succeeds. `Sidebar.svelte` subscribes and calls `loadRoot()` on signal.
- **Why:** Watcher's write-ignore-list suppresses backend-initiated creates. Shared signal store is the project-wide pattern (already used for scroll requests).

### Bug 2: Autocomplete selected-row contrast
- **D-02:** Force brand accent color on selected row via `!important` — purple background + white text. Detail text uses `rgba(255,255,255,0.75)`.
- **Why:** CM6 default theme (`.cm-tooltip.cm-tooltip-autocomplete > ul > li[aria-selected]`) has higher specificity than our override for `background`. Our label override won on color but left CM6's blue background, producing dark-purple-on-blue (low contrast). Solution: stronger override + white text on brand purple is high-contrast and matches UI-SPEC 60/30/10 (accent is 10% reserved for focus).

### Bug 3: Backlinks panel toggle discoverability
- **D-03:** Add a **PanelRight icon button** in the editor topbar (right side, symmetric to the existing left sidebar collapse button). Click toggles `backlinksStore`, aria-label "Backlinks-Panel umschalten", title tooltip.
- **Why:** Keyboard-only discovery fails for new users. A visible toggle matches the existing pattern (left sidebar has a toggle button in `.vc-editor-topbar`).

### Bug 4: Rename-cascade race condition
- **D-04:** Rewrite `update_links_after_rename` to iterate `FileIndex.all_relative_paths()` with **rayon parallel iter**, apply regex `\[\[old_stem(\|alias)?\]\]` to each file's content, and rewrite if matched. Drop the `lg.incoming_for(&old_path)` dependency entirely.
- **Why:** File rename triggers watcher, watcher dispatches `RemoveLinks(old_path)` to the LinkGraph consumer, and by the time `update_links_after_rename` reads `incoming_for(old_path)` it's empty. This is the RESEARCH.md Pattern 6 design (rayon-parallel file rewrite), which the original implementation silently substituted with a graph lookup.
- **Additional:** Add an integration test that renames a file with known backlinks and asserts `updated_files > 0`.

### Bug 5: `[[` + closeBrackets → `[[]]` kills autocomplete
- **D-05:** In `wikiLinkAutocomplete.ts`, remove the unconditional `afterCursor.startsWith("]]")` bailout. Replace with: bail **only** when there's a non-empty `innerText` AND `]]` follows (i.e., cursor is inside a *completed* link being re-triggered — we don't want that). When `innerText === ""` and `]]` follows, still show suggestions.
- **D-06:** When `]]` already follows the cursor (auto-inserted by closeBrackets), `apply` must insert only the filename, not `${name}]]`. Detect with `hasClosingBrackets` flag and branch the `apply` string.
- **Why:** Most natural user flow: type `[[`, expect popup. closeBrackets auto-inserts `]]`. Must handle both cases gracefully.

### Claude's Discretion

- Exact toast-like treatment of the panel toggle button (icon size, hover state)
- Test strategy for Bug 5 (unit test via CompletionContext mock, or integration)

</decisions>
